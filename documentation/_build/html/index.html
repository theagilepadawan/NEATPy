This is the documentation for NEATpy: a transport system conforming to the specification of a transport system specified by the TAPS WG. While written in Python, it utilizes the NEAT codebase with the help of language bindings created by SWIG. In this way this transport system is logically divided in a front-end and back-end; the Python front-end presents a standard conforming API to the end user, while under the hood, uses NEAT to handle all protocol machinery.
